# 项目背景

## 项目概述

随着大语言模型的发展，游戏NPC已经具备自然语言交互能力，但现阶段AI NPC仍然本质上是“被动响应式聊天机器人”。

它们能够回答问题，却无法真正理解自己是谁、身处什么环境、经历过什么事情，以及未来应该如何成长。

玩家获得的是一次次独立的对话，而不是与一个持续存在的角色共同生活。

本项目希望构建一套基于人格、记忆、世界和时间驱动的NPC生命系统，使NPC能够在虚拟世界中持续成长、经历事件、积累记忆并形成稳定人格，从而实现更接近真实生命体的长期互动体验。

------

## 行业现状

当前主流AI NPC方案主要采用：

```text
玩家输入
↓
Prompt组装
↓
LLM生成
↓
角色回复
```

虽然提升了语言自然度，但角色本身仍存在严重缺陷。

无论是游戏NPC还是AI陪伴角色，其智能主要集中于“说什么”，而非“为什么这样说”。

角色缺少对自身、世界和时间的理解，因此无法形成持续成长能力。

------

# 当前痛点分析

## 痛点一：角色世界观不稳定

现有AI角色通常只保存设定文本。

例如：

```text
你是一名酒馆老板
```

但模型并不真正理解：

- 酒馆在哪里
- 城镇是什么样
- 最近发生什么事件
- 自己与其他角色关系如何

因此会出现：

```text
昨天住在王城
今天住在边境
明天又来自海岛
```

角色认知经常发生漂移。

### 本质问题

缺少持续运行的世界模型（World Model）。

------

## 痛点二：角色无法感知当前场景

大多数NPC对话脱离环境。

例如：

玩家站在战场：

```text
村庄已经被攻陷
到处都是尸体
```

NPC却回答：

```text
今天天气真不错
```

因为模型只看到了用户输入。

没有看到：

```text
当前地点
当前时间
周围人物
正在发生的事件
```

### 本质问题

缺少实时场景系统（Scene System）。

------

## 痛点三：角色不会主动推动剧情

现有NPC主要依赖玩家驱动。

玩家不开口：

```text
NPC什么都不会发生
```

导致：

- 世界静止
- 角色静止
- 剧情静止

NPC缺少目标与计划。

### 本质问题

缺少行为驱动系统（Behavior System）。

------

## 痛点四：角色没有自己的生活

玩家离线后：

```text
NPC暂停存在
```

再次上线：

```text
NPC仍然停留在原地
```

现实中的人会：

- 工作
- 睡觉
- 社交
- 产生新经历

而AI NPC不会。

### 本质问题

缺少后台生命模拟系统（Life Simulation）。

------

## 痛点五：记忆缺乏时间组织

当前大部分记忆系统：

```text
向量数据库
+
相似度检索
```

存在问题：

### 时间顺序混乱

NPC可能同时回忆：

```text
三个月前的事情
昨天的事情
十分钟前的事情
```

无法判断先后关系。

------

### 记忆重要性混乱

NPC可能记住：

```text
你喜欢吃苹果
```

却忘记：

```text
你救过他的命
```

------

### 记忆激活错误

在当前情境下不相关的记忆被频繁召回。

导致角色行为异常。

### 本质问题

缺少时间驱动的记忆编排系统（Temporal Memory System）。

------

## 痛点六：人格不会成长

当前人格设定：

```text
勇敢
善良
乐观
```

从创建到删除始终不变。

而真实人格会受到：

- 时间
- 经历
- 关系
- 创伤
- 成功

影响。

例如：

```text
乐观
↓
战争经历
↓
谨慎
↓
背叛经历
↓
冷漠
```

人格应当是动态演化的。

### 本质问题

缺少人格演化系统（Personality Evolution）。

------

# 用户分析

## 核心用户

### RPG玩家

需求：

- 角色拥有成长轨迹
- 剧情能够持续发展
- 关系长期演变

他们期待：

> 与一个活着的人互动，而不是聊天窗口。

------

### 开放世界玩家

需求：

- 世界持续运行
- NPC拥有自己的生活

他们期待：

> 即使我离线，世界仍在运转。

------

### AI陪伴用户

需求：

- 长期陪伴
- 共同经历
- 情感积累

他们期待：

> 角色能够和自己一起成长。

------

# 产品定位

## 产品定义

> 一套基于人格、世界、时间和记忆驱动的**NPC叙事生命模拟系统**，为虚拟角色提供持续存在、场景感知、目标驱动和自主叙事生成能力。

**核心升级**（v1.6）：从"生命模拟"进化为"**叙事生命模拟**"——角色不仅会"活"，还会"讲故事"。每一次日程事件都是一段微型叙事，每一个场景变化都有因果链可追溯。

------

## 当前已实现形态（v1.6）

经过多个版本的迭代，CharacterSeed 已从概念验证进化为一个具备完整叙事闭环的独立系统：

```
┌─────────────────────────────────────────────────────────┐
│                   CharacterSeed v1.6                      │
│                                                           │
│  输入层                                                   │
│  ├─ 一句话/故事文本 → Creation LLM → 角色+世界+场景+目标   │
│  ├─ 玩家对话 → Director+Actor 双LLM管线 → 场景感知回复    │
│  └─ 事件推进 → Director+Actor 事件管线 → 叙事化行为生成   │
│                                                           │
│  世界层（World System）                                   │
│  ├─ World 实体表 → 独立持久化的世界模型                    │
│  ├─ Scene 两层场景树 → 概念场景（大陆/王国）+ 实际场景     │
│  └─ SceneChange 因果链 → 每次变化可追溯"为什么"           │
│                                                           │
│  叙事层（Narrative Engine）                               │
│  ├─ Growth LLM → 基于当日经历创作次日剧情脚本             │
│  ├─ 人格加权决策 → 6维人格影响事件完成方式（成功/失败/…）  │
│  └─ 短期目标驱动 → long_term_goal 分解为可执行小目标       │
│                                                           │
│  数据层（9 张表）                                         │
│  characters · worlds · scenes · scene_changes              │
│  events · conversations · chat_sessions                   │
│  memories · growth_logs                                   │
└─────────────────────────────────────────────────────────┘
```

### 核心能力矩阵

| 能力 | v1.0 MVP | v1.6 当前 |
|------|----------|-----------|
| 角色创建 | 文本描述 → 角色 | 一句话/故事 → 角色 + 世界 + 场景树 + 初始目标 + Day1日程 |
| 世界模型 | world_setting 纯文本字段 | World + Scene + SceneChange 三表，场景树结构化 |
| 角色位置 | `current_state.location` 字符串 | `current_scene_id` 指向具体场景，含完整路径链 |
| 对话交互 | Director+Actor 双LLM | + 场景上下文注入，行为融入环境元素 |
| 日程事件 | 无 | 事件推进轴 + Director+Actor 事件管线 + 人格加权决策 |
| 日程叙事 | 无 | 叙事脚本而非待办清单，场景融入、微型故事弧线 |
| 目标系统 | long_term_goal 文本 | + short_term_goals 数组 + 双迭代者（Growth全量 + 角色微调） |
| 世界演化 | 无 | SceneChange 因果链，角色行动改变场景 |
| 可解释性 | 基础 | Director 决策摘要 + Actor 叙事标签 + 场景环境面板 |

------

## 核心定位

不是：

```text
AI聊天系统
```

也不是：

```text
AI NPC系统
```

而是：

```text
NPC叙事生命模拟系统
```

**"叙事生命"的含义**：角色不只被动响应——它拥有世界（Where）、目标（Why）、日程（What），并且每个行动都是一段可读的故事。Growth LLM 不是"状态更新器"而是"编剧"——它观察角色的一天，然后规划下一章的剧情。

------

## 差异化定位

传统方案：

```text
用户输入
↓
LLM
↓
输出
```

人格方案：

```text
用户输入
↓
人格
↓
记忆
↓
LLM
↓
输出
```

我们的方案（v1.6 当前）：

```text
用户输入 / 事件触发
↓
世界系统（World + Scene 树 + SceneChange 因果链）
↓
场景上下文（位置路径 + 相邻场景 + 最近变化）
↓
目标系统（long_term_goal → short_term_goals → 日程对齐）
↓
记忆编排（三层记忆：短期 + 长期 + 知识库）
↓
人格加权（6维人格 → 事件完成概率分布 + 行为倾向）
↓
Director 注意力聚焦（emotion / goal / style / focus_memories）
↓
Actor 叙事生成（场景感知的动作 / 表情 / 语言）
↓
输出（叙事化行为描述 / 对话回复）
↓
经历记录（Event / Memory / Conversation）
↓
Growth 编剧（观察经历 → 人格演化 + 新记忆 + 次日剧情脚本）
↓
世界迭代（SceneChange → 场景演化）
```

------

# 一句话产品定位

> **构建拥有世界感、时间感和叙事成长能力的AI数字生命NPC系统。让NPC不再只是会说话——它们站在具象的场景中，拥有可追溯的因果链，每一天都是一个新的故事章节。**

------

> 当前版本 v1.6：场景感知叙事生命模拟。World + Scene + SceneChange 三层世界模型、Director+Actor 双LLM叙事管线、Growth 编剧系统、短期目标驱动。

> 导演不再负责决策，而负责聚焦。

这个区别非常大。

# 后续版本实现计划

> v1.6 已实现世界系统具象化、场景感知叙事、短期目标系统。以下为后续迭代方向。

---

## 一、立即优化（v1.6.1）

针对 v1.6 的体验收尾和叙事质量提升。

### 1. 前端世界面板独立展示

**现状问题**：
- 当前场景信息散落在对话页和事件面板的折叠区中，用户需展开才能看到
- 世界状态、场景变化时间轴、目标进度缺少独立汇总页面

**优化方案**：
- 新增"🌍 世界"独立页面：场景树可视化、场景变化时间轴、相邻场景探索地图
- 对话页顶部常驻轻量场景指示器（当前位置 + 路径面包屑），不占折叠空间

### 2. 日程叙事质量持续优化

**现状问题**：
- v1.6.fix 叙事增强后日程质量显著提升，但 LLM 在场景信息不完整时仍有退化可能
- 事件类型分布中 scene_event 和 character_initiative 占比需要持续监控

**优化方案**：
- Growth prompt 增加"如果场景信息不足，基于角色习惯和目标合理构建场景"的兜底
- 事件类型强制多样化约束：每轮 schedule 中 non-schedule_action 事件 ≥ 1 条
- 考虑加入后处理校验：检测 content 长度 < 20 字则请求重试

### 3. 场景变化在前端可视化

**现状问题**：
- SceneChange 数据已完整存储在数据库中，但前端仅在折叠面板中显示最近变化文本
- 用户无法直观感知"世界在变"

**优化方案**：
- 世界页面增加时间轴组件：每一天的场景变化按卡片排列
- 变化类型用颜色区分（character_driven=蓝, external=橙）
- 支持回溯查看历史任一天的世界状态快照

---

## 二、计划实现（v1.7+）

### 1. 叙事游戏模式（原 v1.3）

**目标**：从"被动对话"升级为"主动叙事驱动"，Director 编排多步自主场景，在关键节点暂停等待玩家选择。

**核心设计**：
- **GameDirector**：现有 Director 的上层编排器——它调用 Director（注意力）和 Actor（行为），自身只负责"叙事推进策略"
- **Game Loop**：`GameDirector.plan_scene() → 叙事序列执行循环（N步）→ 决策点暂停 → 玩家选择 → 继续推进`
- **固定选项交互**：降低认知负荷，保证叙事聚焦，每个选项由 GameDirector 基于上下文构建
- **与现有模式共存**：自由对话模式 + 叙事冒险模式，通过 toggle 切换

### 2. 同一世界多角色联动（原 v1.2）

**目标**：多个角色在同一个世界中共存，相互感知和互动。

**计划内容**：
- 角色创建时可选加入已有世界（World 表已支持 N:1）
- 新增 Relationship 表：角色间关系类型 + 关系值（0–100）
- Growth 分析时可生成跨角色联动事件
- 统一 WorldTime 管理所有角色时间推进

### 3. 多模态输入输出

**目标**：突破纯文本交互限制。

**计划内容**：
- 语音输入（Whisper API）+ 语音输出（Edge-TTS / ElevenLabs）
- 虚拟形象：基于角色描述生成立绘，根据 emotion 动态切换表情
- 场景背景图：基于 world_setting 和 current_scene 生成场景图
