## 🔑 关键交互流程：发送消息→流式接收→显示

涉及文件按调用顺序：

1. **用户操作**：`web/src/views/ChatView.vue` 的 `<textarea>` + `onSend()` 
2. **业务编排**：`web/src/composables/useChat.ts` 的 `send()`（插入 user 消息 + 占位 assistant 消息）
3. **API 调用**：`web/src/api/index.ts` 的 `chat.streamSend()`（`fetch + ReadableStream` 解析 SSE）
4. **后端管线**：`backend/main.py` 的 `/api/chat/stream` → `backend/modules/interaction.py` 的 `run_stream()`
5. **事件回调**：
   - `thinking` → 更新占位消息的 `thinking_phase` + `thinking_message`
   - `meta` → 写入 `sessionId` + `sessionTitle` + 提前显示 emotion
   - `speech` → 增量追加到占位 `content`（打字机）
   - `done` → 移除占位，替换为完整 `ChatResponse`
6. **持久化**：用户选中角色保存在 `localStorage: cs.activeCharacterId`

## 🔑 关键交互流程（送消息 → 显示）

1. **用户输入** → [ChatView.vue](file:///c:/Users/biren/Documents/trae_projects/luyan/CharacterSeed/web/src/views/ChatView.vue) `onSend()`
2. **业务编排** → [useChat.ts](file:///c:/Users/biren/Documents/trae_projects/luyan/CharacterSeed/web/src/composables/useChat.ts) `send()`（插入 user 消息 + 占位 assistant）
