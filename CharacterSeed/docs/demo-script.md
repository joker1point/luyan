## 📜 7 步逐字稿

### Step 1 · T+0s · 新建角色"小雅"

| 项 | 内容 |
|---|---|
| **UI 操作** | 1) 点击左侧"创建新角色" 2) 填写：名字"小雅" / 设定"温柔的高中语文老师，喜欢不加糖的拿铁" 3) 点击"创建" |
| **界面显示** | 角色卡片出现：头像 + "小雅 · 32岁 · 高中语文老师 · 成都" |
| **数据浮窗** | `Memory 数量: 0` / `jiwen state: 未初始化` |
| **旁白**（3s） | "我们刚建了一个角色'小雅'。它的记忆库是空的。" |
| **同时讲解** | "请大家看右上角——Memory 数量是 0，jiwen 状态未初始化。" |

---

### Step 2 · T+8s · 第 1 句对话

| 项 | 内容 |
|---|---|
| **UI 操作** | 在聊天框输入："我喜欢不加糖的拿铁" → 点击发送 |
| **加载时长** | 1.2 秒（Director + Actor 双 LLM） |
| **角色回复** | "原来你不爱甜味呀 ☕ 我自己倒喜欢加一点焦糖，不过你的口味我记下了。" |
| **数据浮窗（自动刷新）** | `Memory 数量: 1` / `最近记忆: "用户喜欢不加糖的拿铁"` / `importance: 8, theme: taste` |
| **旁白**（2s） | "它告诉你'我记下了'——这不是客套，是真的在记。" |

---

### Step 3 · T+15s · 第 2 句对话

| 项 | 内容 |
|---|---|
| **UI 操作** | 输入："今天心情不好" → 发送 |
| **角色回复** | "怎么了？愿意说说吗？我今天也批改作业改到有点烦，不过听你聊聊天会好一些。" |
| **数据浮窗（自动刷新）** | `jiwen 5 轴: connection 0.62 / valence -0.18 / arousal 0.10`（5 轴实时显示） |
| **旁白**（2s） | "它也表达了'我也有点烦'——这是**jiwen 5 轴情感引擎**在工作。" |
| **同时讲解** | "看 connection 0.62——它对你产生了'连接感'。这不是脚本，是数学。" |

---

### Step 4 · T+25s · 关闭浏览器 · "7 天后"

| 项 | 内容 |
|---|---|
| **UI 操作** | 1) 关闭浏览器 2) 系统时间调到 7 天后 3) 重新打开浏览器 4) 登录同一账号 5) 进入"小雅"聊天页 |
| **加载时长** | 0.8 秒（缓存已加载） |
| **界面显示** | 角色卡片 + 聊天框为空 + 顶部小字提示"上次对话：7 天 3 小时 12 分前" |
| **旁白**（3s） | "现在让我们快进 7 天。" |
| **同时操作** | 主持人把系统时间从 `2026-06-20 14:00` 改到 `2026-06-27 17:12`（实操时使用录屏切镜头） |
| **关键视觉** | 时间戳 7 天前的浮窗要放大居中，停留 1.5 秒 |

> ⚠️ **演示技巧**：实际路演时**不要真等 7 天**，直接切录屏 / 用预录视频。主持人用嘴播报"7 天后"即可。

---

### Step 5 · T+35s · "你还记得我吗？"

| 项 | 内容 |
|---|---|
| **UI 操作** | 在聊天框输入："你还记得我吗？" → 发送 |
| **加载时长** | 1.5 秒（Director 检索长期记忆 + Actor 生成） |
| **角色回复** | "当然记得呀 ☕ 你喜欢的还是不加糖的拿铁吧？今天又加班到这个点？上次你说周三最累……" |
| **数据浮窗（自动刷新）** | `检索命中: 1 条 Memory` / `strength: 0.72（衰减后仍可检索）` / `时间戳: 7 天 3 小时前` / `jiwen connection: 0.74 ↑` |
| **旁白**（5s） | "**她记得。** 7 天、3 个小时、12 分钟——精确到分钟。拿铁的口味、上次你说的话——一字不差。" |
| **同时讲解** | "看 strength 0.72——按 Ebbinghaus 曲线已经衰减到 72%，但**仍然高于检索阈值**。这就是我们 90 天只花 3k tokens/轮的秘密。" |

> 🔑 **全路演最关键 5 秒**：角色回复与 7 天前输入一字不差，**这是评委最买账的瞬间**。

---

### Step 6 · T+50s · 切后台 · 展示数据库

| 项 | 内容 |
|---|---|
| **UI 操作** | Alt+Tab 切到 Terminal + DB Browser 窗口 |
| **DB Browser 显示** | `characterseed.db` → `memories` 表 → 展示一条记录： |
|  | `id=1` `character_id=1` `content="用户喜欢不加糖的拿铁"` `importance=8` `theme="taste"` `strength=7.2` `forgotten=0` `created_at="2026-06-20 14:00"` |
| **Terminal 显示** | `curl /api/characters/1/memories` 输出 JSON |
| **旁白**（5s） | "不是黑盒。我们把数据库打开——这就是它'记住'的内容：主题、重要性、强度、创建时间。" |
| **同时讲解** | "Soft delete，不是 hard delete。这条记忆永远可以恢复。GDPR 友好。" |

---

### Step 7 · T+58s · 回到角色 · 收尾

| **旁白**（2s） | "ChatGPT 让 AI 学会'说话'。我们让 AI 学会'记住'。" |
| **金句浮窗**（全屏 1s） | **"90 天只花 3k tokens/轮"** |

## 📊 演示数据浮窗规范

### 浮窗 1：Memory 详情卡
```json
{
  "memory_id": 1,
  "character_id": 1,
  "content": "用户喜欢不加糖的拿铁",
  "importance": 8,
  "theme": "taste",
  "type": "preference",
  "strength_initial": 10.0,
  "strength_current": 7.2,        // 衰减 7 天后
  "half_life_days": 90,            // taste 类
  "recall_count": 1,
  "forgotten": false,
  "created_at": "2026-06-20 14:00:00"
}
```

### 浮窗 2：jiwen 5 轴情感状态
```json
{
  "connection": 0.74,    // 主动想起 → 高
  "pride":      0.32,    // 中性
  "valence":   -0.18,    // 受用户影响轻微下降
  "arousal":    0.10,    // 中性
  "immersion":  0.85     // 沉浸感高
}
```

### 浮窗 3：成本对比
```
单轮 LLM 调用:  2 次   (Director + Actor)
Token / 轮:     3,012 tokens  (vs 传统 30,000)
响应延迟:       1.5s (P95)
```
**大数字浮窗**（占屏幕 1/3）：**省 90% token**
