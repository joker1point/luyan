### P5 — 4 大创新（60s）— 全场最重要

**画面**：4 个并列卡片，每个包含 标题 + 价值主张 + 一行金句

#### 创新 1：Ebbinghaus 主题差异化衰减
- **公式**：`strength(t) = initial × 0.5^(t / half_life) × (1 + 0.1 × log(1+recall))`
- **关键数字**：identity 半衰期 **365 天** vs todo 半衰期 **1 天**
- **金句**：
  > "自动遗忘噪声，永久保留核心。"

#### 创新 2：自适应摘要触发器
- **不是** 固定 50 条触发
- **而是** 4 条件动态：消息数 ≥ 100 / 消息数 ≥ 20 且遗忘率 > 30% / 消息数 ≥ 20 且时间 > 7 天 / 手动
- **金句**：
  > "低 importance 角色快速滚，高 importance 角色慢慢积累。"

#### 创新 3：jiwen 5 轴情感引擎（Node → Python 纯算法移植）
- **5 轴**：connection / pride / valence / arousal / immersion
- **杀手锏**：connection 越高，NPC **主动发起对话**——从"被动应答"到"主动陪伴"
- **金句**：
  > "AI 不再等你问，AI 想找你聊。"

#### 创新 4：零模板注入策略
- **做法**：5 轴 / 世界 / 关系网塞进 `current_state._xxx` 子字段
- **效果**：换 LLM 0 迁移成本，模型升级 0 改造
- **金句**：
  > "换 GPT / Claude / Qwen？我们改一行。"

**话术收尾**：
> "这 4 个创新点，**单独看都能抄，组合起来极难抄**。"

### P6 — 成本杀手（45s）— 投资人最关心

**画面**：双柱状图

| 场景 | 传统方案 | CharacterSeed | 节省 |
|---|---:|---:|---:|
| 单轮 LLM 调用 | 2 次（Director+Actor）+ 1 次重述 | **2 次** | - |
| Token/轮 | 30k | **3k** | **90%** |
| 摘要调用 | 固定 50 条触发 | 20-100 自适应 | **40%** |
| 记忆提取 | 同步阻塞 | **异步 daemon** | **0ms 阻塞** |
| 背景线程数 | 1 | 3（jiwen+decay+summary 并行） | - |

**大数字浮窗**（占据屏幕中央）：
> **"省 90% token"**
> **"省 40% 摘要调用"**
> **"响应延迟 < 800ms"**

**话术**：
> "我们用 3 件事省钱：  
> ① **衰减压 token**（-90%）  
> ② **异步不阻塞**（响应 < 800ms）  
> ③ **单例 LLM 复用**（Director 顺便提取记忆，0 额外调用）"

## 🎤 现场提问预案

### Q1："为什么不是直接用 Mem0 / LangChain Memory？"
**答**：他们没有"衰减"。我们的 Ebbinghaus 主题差异化是核心壁垒——Mem0 永远把 30 天前的'你今天想喝什么'和你初恋的回忆放在同等权重，我们不会。

### Q2："用户隐私怎么办？"
**答**：SQLite 本地存储 + 可选 AES-256 加密；关键记忆"软删除"不物理删除，用户可恢复；所有数据不出端。

### Q3："成本真的这么低吗？"
**答**：100 万 MAU × 30 天 × 10 轮/天 × 3k tokens = 9 亿 token/天 → qwen 长文本约 **$10k/月**；比纯全量上下文省 90%。

### Q4："竞品 Replika / Character.AI 不是做了吗？"
**答**：他们用全量上下文（贵 10x），我们用衰减 + 摘要。**降本 90% 是核心优势**——同样效果，成本是 1/10，毛利高 9 倍。

### Q5："技术壁垒会不会被复制？"
**答**：单点易抄，系统难抄。Ebbinghaus 公式公开，但我们 + 主题差异化 + 自适应摘要 + jiwen 5 轴 + 零模板注入是**协同护城河**。抄一个少一个，组合起来 2 年工程量。

### Q6："怎么冷启动？"
**答**：先做游戏 NPC SDK（3-6 个月）→ 1-2 个爆款植入 → 情感陪伴 C 端 → 教育 / 客服。路径短、付费意愿强。

### Q7："团队几个人？全职？"
**答**：核心 2 人，互补搭档。**文泓发**负责全栈设计 + 技术选型（FastAPI / React / SQLite / jiwen 移植），**黄天豪**是队长，负责产品方向 + 市场洞察 + 运营分发。准备 6 个月内按需扩到 3-4 人。

### Q8："融资计划？"
**答**：种子轮 200-500 万 RMB，6 个月 run rate，产品-市场匹配（PMF）跑通，A 轮 3000 万+。
