#### 创新 1：Ebbinghaus 主题差异化衰减
- 公式：`strength(t) = initial × 0.5^(t / half_life) × (1 + 0.1 × log(1+recall))`
- 关键数字：identity 半衰期 **365 天** vs todo 半衰期 **1 天**
- 金句：**"自动遗忘噪声，永久保留核心。"**

#### 创新 2：自适应摘要触发器
- **不是** 固定 50 条触发
- **而是** 4 条件动态：消息数 ≥ 100 / 消息数 ≥ 20 且遗忘率 > 30% / 消息数 ≥ 20 且时间 > 7 天 / 手动
- 金句：**"低 importance 角色快速滚，高 importance 角色慢慢积累。"**

#### 创新 3：jiwen 5 轴情感引擎（Node → Python 纯算法移植）
- 5 轴：connection / pride / valence / arousal / immersion
- 金句：**"AI 不再等你问，AI 想找你聊。"**

#### 创新 4：零模板注入策略
- 把 5 轴 / 世界 / 关系塞进 `current_state._xxx` 子字段
- 金句：**"换 GPT / Claude / Qwen？我们改一行。"**

## 🎤 现场提问预案（8 个高频问题）

### Q1："为什么不是直接用 Mem0 / LangChain Memory？"
**答**：他们没有衰减。Ebbinghaus 主题差异化是核心壁垒——Mem0 永远把 30 天前的"你今天想喝什么"和初恋回忆放在同等权重，我们不会。

### Q2："用户隐私怎么办？"
**答**：SQLite 本地存储 + 可选 AES-256 加密；关键记忆"软删除"不物理删除，用户可恢复；所有数据不出端，GDPR 友好。

### Q3："成本真的这么低吗？"
**答**：100 万 MAU × 10 轮/天 × 3k tokens = 9 亿 token/天 → qwen 长文本约 **$10k/月**，比纯全量上下文省 90%。

### Q4："竞品 Replika / Character.AI 不是做了吗？"
**答**：他们用全量上下文（贵 10x），我们用衰减 + 摘要。降本 90% 是核心优势——同样效果，成本 1/10，毛利 9 倍。

### Q5："技术壁垒会不会被复制？"
**答**：单点易抄，系统难抄。Ebbinghaus + 自适应摘要 + jiwen 5 轴 + 零模板注入是协同护城河，组合 2 年工程量。

### Q6："怎么冷启动？"
**答**：游戏 NPC SDK（3-6 月）→ 1-2 个爆款植入 → 情感陪伴 C 端 → 教育/客服。路径短、付费意愿强。

### Q7："团队几个人？全职？"
**答**：核心 **2 人互补**。

## 🛠 技术护城河（深度展开）

### 1. Ebbinghaus 主题差异化衰减

**为什么是"主题差异化"？**
- 通用衰减一视同仁，**错的**
- 人的记忆是分层的：身份层（"我是谁"）衰减极慢，琐事层（"今天午饭吃啥"）衰减极快
- 我们给每条记忆打 theme 标签（identity / event / preference / todo / misc），不同主题配不同半衰期

**实测数据**：
- `identity` 主题：half_life=365 天，30 天后 strength 仍保留 95%
- `todo` 主题：half_life=1 天，30 天后 strength 仅 0.0001
- 这才是"记得你"与"忘掉噪声"兼得

### 2. 自适应摘要触发器

**4 条件动态触发**（不是固定 50 条）：
1. 消息数 ≥ 100
2. 消息数 ≥ 20 **且** 遗忘率 > 30%
3. 消息数 ≥ 20 **且** 时间 > 7 天
4. 手动触发（用户可主动压缩）

**优势**：
- 高 importance 角色：慢慢积累，7 天才触发
- 低 importance 角色：快速滚动，2 天就触发
- 节省 40% 摘要调用成本

### 3. jiwen 5 轴情感引擎

**5 轴独立建模**：
- `connection`（与你亲密度） — 高 = 主动发起对话
- `pride`（自尊水平） — 高 = 不愿示弱
- `valence`（正负情绪） — 高 = 开心，低 = 沮丧
- `arousal`（激活度） — 高 = 兴奋，低 = 平静
- `immersion`（沉浸度） — 高 = 愿意深入话题

**应用**：
- 高 `connection` 角色：早安问候 / 主动找话题
- 高 `valence` + 低 `arousal`：平静表达
- 低 `valence` + 高 `arousal`：激动 / 愤怒

**移植细节**：
- 原版是 Node.js（ClaraShafiq/jiwen）
- 我们用 Python 重写 + 适配 FastAPI 异步架构
- 数学漂移 + 触发器逻辑纯函数化，0 网络依赖

### 4. 零模板注入策略

**核心思想**：不让 LLM "看模板"，而是把"上下文子字段"塞进角色的 `current_state` 字典。

```python
character.current_state = {
    "mood": "happy",          # 业务字段
    "_jiwen": {                # 注入字段（下划线开头）
        "connection": 0.62,
        "valence": 0.45,
    },
    "_world": {                # 注入字段
        "season": "spring",
        "day": 61,
    },
    "_relationships": [        # 注入字段
```
